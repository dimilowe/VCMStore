What you actually want
Either of these is fine; both are better than inline styles:
Option A – Fix .text-gradient globally (best)
In your global CSS where .text-gradient is defined, update it so children don’t inherit the transparent fill:
/* existing gradient */
.text-gradient {
  background-image: linear-gradient(90deg, #ec4899, #f97316);
  -webkit-background-clip: text;
  color: transparent;
  -webkit-text-fill-color: transparent;
}

/* new: make children inherit normal color */
.text-gradient * {
  -webkit-text-fill-color: currentColor;
  color: inherit;
}
Now only the element with .text-gradient is “fancy text”; anything inside it can render normally.
Option B – Scoped reset for tool pages
In a CSS file that’s imported by your tool layout, not global marketing, add:
.tool-page-root {
  -webkit-text-fill-color: currentColor;
}

/* if needed, be more explicit */
.tool-page-root * {
  -webkit-text-fill-color: currentColor;
}
Then make sure your tool layout wraps content with className="tool-page-root".
This keeps marketing gradient behavior but neutralizes it inside tools.

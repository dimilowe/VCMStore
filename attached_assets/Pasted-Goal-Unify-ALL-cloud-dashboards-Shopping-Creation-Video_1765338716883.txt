Goal: Unify ALL cloud dashboards (Shopping, Creation, Video, Writing, File/Data, Intelligence, Music, Growth/Distribution, etc.) under a single, beautiful white-theme Adobe-style layout, while strictly respecting the existing architecture and Editing Rules.

This is a PURELY VISUAL + LAYOUT refactor:
- Do NOT change business logic, renderer logic, CMS contracts, or entitlements.
- Do NOT change which tools belong to which cloud.
- Only change how the dashboards LOOK and how their sections are laid out.

GLOBAL EDITING RULES (DO NOT VIOLATE)

Forbidden (ABSOLUTELY READ-ONLY)
- components/renderer/**
- lib/vcmRenderer/**
- lib/vcm-renderer/**
- Any file containing ANY of these identifiers:
  - Renderer
  - renderEngine
  - VCMRenderer
  - resolveConfig
  - resolveCmsObject
- app/globals.css
- lib/db.ts
- Engine registries: lib/engineRegistry.ts, data/toolsRegistry.ts
- Any low-level CMS resolver or routing code

The codebase is LIGHT-THEME ONLY:
- Do NOT use `dark:` Tailwind variants anywhere.
- Do NOT introduce new theme toggles or dark mode.

Allowed (YOU MAY EDIT / CREATE)
- app/clouds/** (all cloud dashboard pages)
- components/clouds/**
- components/cards/**
- components/ui/** (for generic presentational components)
- Any NEW presentational components living under components/clouds/ or components/cards/

Styling constraints:
- All styling must be done with Tailwind classes in TSX.
- No edits to app/globals.css or Tailwind config.

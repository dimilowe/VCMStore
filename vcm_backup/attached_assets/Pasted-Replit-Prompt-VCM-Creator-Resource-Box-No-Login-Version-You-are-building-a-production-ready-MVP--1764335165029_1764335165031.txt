Replit Prompt – VCM Creator Resource Box (No-Login Version)
You are building a production-ready MVP web app called VCM Creator Resource Box.
This app is a free, simple link box / resource card generator that creators can use with no login:
They fill out a form.
They get a unique URL + iframe embed.
That URL lives on our domain and routes traffic through our ecosystem.
Some items can be our internal free tools/freebies, some can be external links.
There is no authentication for creators.
All tools and boxes are publicly accessible.
1. Tech Stack
Use this stack:
Backend: Node.js + Express
Database: SQLite using Prisma ORM
Frontend: EJS templates (server-rendered)
Styling: Tailwind CSS (precompiled to a static CSS file)
Deployment: Standard Replit Node server
Keep it simple and readable.
2. Core Data Models
We only need:
Box – one generated resource card
BoxItem – each link inside the Box
InternalResource – pre-defined free tools / freebies we host
Prisma schema:
model Box {
  id          Int        @id @default(autoincrement())
  slug        String     @unique
  title       String
  subtitle    String?
  accentColor String?    // optional hex color
  createdAt   DateTime   @default(now())

  items       BoxItem[]
}

model BoxItem {
  id                 Int              @id @default(autoincrement())
  box                Box              @relation(fields: [boxId], references: [id])
  boxId              Int
  label              String           // visible text
  description        String?          // optional subtext
  type               BoxItemType
  externalUrl        String?          // used if type = external
  internalResource   InternalResource? @relation(fields: [internalResourceId], references: [id])
  internalResourceId Int?
  position           Int              // order (1–4)
}

model InternalResource {
  id          Int              @id @default(autoincrement())
  name        String
  slug        String   @unique
  url         String            // absolute URL to our existing tool/freebie
  category    String            // "tool" | "freebie"
  shortTag    String?           // e.g. "Free Tool", "Template"
  description String?

  boxItems    BoxItem[]
}

enum BoxItemType {
  external
  internal_tool
  internal_freebie
}
Seed InternalResource with some of our tools/freebies, e.g.:
Word Counter – category tool
Image Compressor – tool
Self-Love Affirmation Pack – freebie
Creator Hook Template Pack – freebie
3. Routes & Flow
Public Landing / Generator
GET /
Show a simple landing page that:
Explains what the Creator Resource Box is.
Shows one demo Box (you can use a seeded Box).
Includes a Box Generator Form with:
Box title (required)
Subtitle (optional)
Accent color (optional hex input, default to a nice purple)
Up to 4 BoxItems:
For each item:
label (text)
description (optional)
type (radio: external / internal_tool / internal_freebie)
If external: a URL input
If internal_*: a dropdown populated from InternalResource
The form can show 4 rows for items by default; allow empty labels to mean “unused item”.
POST /create-box
Process the form.
Validate:
title required.
At least 1 valid item (non-empty label).
Max 4 items.
For each item:
If type = external → require a valid URL.
If type = internal_tool or internal_freebie → require a valid internalResourceId from DB and that it matches the chosen type category.
Create Box row.
Create BoxItem rows with sequential position (1–4 based on order).
Generate a slug for the Box:
e.g. slug = nanoid() or box-<short-random-string>.
Redirect to GET /box/:slug/generate-success.
GET /box/:slug/generate-success
Fetch the Box and its items.
Show:
A preview of the Box.
A public link:
https://YOUR_REPL_URL/box/:slug
An iframe embed snippet:
<iframe
  src="https://YOUR_REPL_URL/embed/box/:slug"
  style="border:0;width:100%;max-width:420px;height:260px;overflow:hidden;border-radius:16px;"
  loading="lazy"
  referrerpolicy="no-referrer-when-downgrade"
></iframe>
A “Create another Box” button back to /.
Creators don’t need accounts. They generate, copy link/embed, and leave.
Public Box Display
GET /box/:slug
Fetch Box by slug with its items sorted by position.
Render a full-page, clean card view:
Centered card
Top:
small meta text: “Creator Resource Box”
box title
subtitle (if any)
Middle:
up to 4 link cards/buttons
if item is internal and InternalResource.shortTag exists, show small tag badge (e.g. “Free Tool”)
Bottom:
Tiny text: “Powered by VCM Suite · Upgrade to Nudg” (link placeholder for Nudg).
This should look nice even if visited directly on desktop or mobile.
GET /embed/box/:slug
Similar to /box/:slug but:
No huge margins; more compact.
No extra navigation or page chrome.
Slightly smaller typography.
Footer can just be a tiny line: “Powered by VCM Suite”.
Ensure you do not send X-Frame-Options: DENY so iframes work.

6. No Auth, No Analytics (For Now)
No login screens.
No user accounts.
No dashboards.
Everything is:
Public generator at /.
Public boxes at /box/:slug.
Public embeds at /embed/box/:slug.
We can add analytics, auth, and a full Nudg upgrade flow later. For now the priority is:

Make it look clean and professional.
Make links to our tools/freebies work.
Make the embed snippet copy-paste friendly.
